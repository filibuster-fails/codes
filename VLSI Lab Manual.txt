Design Information and Bock Diagram:
A 4-bit shift and add multiplier is a digital circuit designed to multiply two 4-bit binary numbers by sequentially shifting and adding, mimicking the long multiplication method used in arithmetic. It consists of a multiplicand, which is the number being multiplied, and a multiplier, which dictates how many times the multiplicand is added. The process begins by storing both numbers in separate registers and initializing a product register to zero. The multiplication proceeds bit by bit, starting from the least significant bit (LSB) of the multiplier. If the current bit is 1, the multiplicand is added to the product register; if the bit is 0, no addition is performed. After each step, the multiplicand is left-shifted to align with the next bit of the multiplier, and the multiplier is right-shifted to bring the next bit into position. This process continues for four cycles, as each bit of the multiplier is processed sequentially.

Design Information and Bock Diagram:
An Arithmetic Logic Unit (ALU) is a fundamental component of a processor that performs both arithmetic and logical operations. It is a combinational circuit that takes input operands, processes them based on a control signal, and produces an output. In this experiment, we design a 32-bit ALU that supports four logical operations (AND, OR, XOR, NOR) and four arithmetic operations (Addition, Subtraction, Multiplication, and Division) using a behavioral modeling approach in Verilog. The selection of the required operation is controlled by a 3-bit ALU_Sel signal, allowing the ALU to perform different computations based on its value.
The ALU consists of several key components, including a multiplexer (MUX) to select the desired operation, an arithmetic circuit to handle addition, subtraction, multiplication, and division, and a logical circuit to process AND, OR, XOR, and NOR operations. Additionally, the ALU implements a Zero Flag, which is set when the output result is zero. This flag is useful for decision-making processes in microprocessors, where conditional branching may depend on the result of an ALU operation.
The Verilog design is implemented using behavioral modeling, which allows the use of case and if statements for defining the ALU's operations. This approach makes the design flexible, readable, and easy to modify. Behavioral modeling abstracts the internal logic and focuses on functionality, making it an efficient choice for ALU implementation. The ALU operations are encoded as follows: 000 for AND, 001 for OR, 010 for XOR, 011 for NOR, 100 for addition, 101 for subtraction, 110 for multiplication, and 111 for division. The division operation is

carefully implemented to handle cases where the denominator is zero, ensuring error-free computation.
To verify the correctness of the ALU, a testbench is written in Verilog, where multiple test cases are executed to validate the functionality of each operation. Once verified, the ALU is synthesized by targeting an appropriate library while setting area and timing constraints. The synthesized netlist is analyzed to extract key performance metrics, including area, power consumption, and delay. Furthermore, the critical path is identified to determine the slowest combinational logic path, which impacts the ALU's overall performance.
This experiment provides insights into digital design, combinational circuits, and hardware description languages (HDL). The implementation of a 32 -bit ALU is essential in understanding the working principles of microprocessors and their arithmetic logic capabilities. By synthesizing the design and analyzing key metrics, we gain practical knowledge about optimizing hardware for efficiency and speed, which is crucial in modern computing systems.

What are flip flops in electronics? A flip-flop is an electronic circuit that can store single-bit binary data either logic 0 or logic 1. Basically, a flip flop is a Bistable multivibrator that changes its output depending on the input. Flip Flops are of two types edge triggered, and level triggered. State of an Edge triggered flip flop changes during the positive or negative edge of a clock cycle. Whereas in the case of level-triggered flip flop output can change during high or low clock duration. Level triggered flip flops are called latch, they are transparent because when they are enabled their output becomes the same as the input.

Theory:
A D Flip-Flop (Data or Delay Flip-Flop) is a sequential circuit used for storing and transferring data in digital systems. It operates based on a clock signal, capturing the value of the D (data) input at the moment of a clock transition and holding this value until the next clock edge. The Q (output) follows the D input on the rising or falling edge of the clock, ensuring reliable data storage. An optional reset (RST) input can be used to force the output Q to 0, regardless of the clock state. The truth table of the D Flip-Flop shows that when the clock edge occurs, the output Q takes the value of D, while its complement Q' takes the opposite value. This behavior makes the D Flip-Flop a fundamental building block in memory units, shift registers, counters, and finite state machines. It can be implemented using logic gates, as a modification of the SR Flip- Flop, or programmed using Verilog/VHDL for FPGA and ASIC designs. D Flip-Flops are commonly used in clock synchronization, data storage, and frequency division applications. Their ability to store and transfer data reliably makes them essential in modern digital electronics.

Theory: An SR Flip-Flop, also known as a Set-Reset Flip-Flop, is one of the fundamental building blocks of sequential logic circuits. It is a bistable device, meaning it has two stable states and can store one bit of binary information (either 0 or 1). The flip-flop has two inputs, labeled S (Set) and R (Reset), along with two outputs Q and QŻ (Q bar), where QŻ is the complement of Q. The operation of the SR Flip-Flop is straightforward: when the Set input (S) is activated (logic 1) and Reset (R) is at logic 0, the output Q is set to 1. Conversely, when R is 1 and S is 0, the output Q is reset to 0. If both S and R are at logic 0, the flip-flop maintains its previous state, functioning as a memory element. However, if both inputs are set to logic 1 simultaneously, it creates an invalid or undefined condition, especially in the NOR gate-based implementation, as both outputs Q and QŻ would become 0, violating the rule of being complementary.
The SR Flip-Flop can be built using NOR or NAND gates, and it can operate either asynchronously, without a clock signal, or synchronously, with a clock input that controls the state change. Due to its simplicity, the SR Flip-Flop is widely used in applications such as data storage, control systems, switch debouncing, and small-scale memory designs. Despite its limitations, particularly the undefined state when both inputs are high, it serves as the foundation for more complex flip-flops like D, JK, and T flip-flops.

Verilog code:
module sr_flipflop( output reg q, qbar, input s, r, clk
);
always @(posedge clk) begin if (s == 1 && r == 0) begin
q <= 1;
  qbar <= 0; end
else if (s == 0 && r == 1) begin q <= 0;
  qbar <= 1; end
else if (s == 0 && r == 0) begin q <= q; // Hold previous state qbar <= qbar;
end
else if (s == 1 && r == 1) begin q <= 1'bx; // Undefined state qbar <= 1'bx;
  end end
endmodule

Theory:

The JK Flip-Flop is an advanced and versatile type of flip-flop that eliminates the invalid state encountered in the SR Flip-Flop. Named after its inventor Jack Kilby, the JK Flip-Flop has two inputs, labeled J and K, along with two outputs Q and QŻ . It also operates based on a clock signal, making it a synchronous sequential circuit. The JK Flip-Flop functions similarly to the SR Flip-Flop in most input conditions: when J = 1 and K = 0, it sets the output Q to 1; when J = 0 and K = 1, it resets Q to 0; and when both J and K are 0, the output retains its previous state (no change). However, the key advantage of the JK Flip-Flop is its behavior when both inputs J and K are set to 1 - instead of producing an invalid state like in SR Flip-Flop, the JK Flip-Flop toggles its output. That means if the current state of Q is 1, it becomes 0, and vice versa.

This toggle feature makes the JK Flip-Flop highly useful in counters, frequency dividers, and control applications. It is considered a universal flip-flop because it can be configured to behave like an SR, D, or T flip-flop by appropriately connecting its inputs. Internally, it typically consists of logic gates arranged in a feedback configuration that ensures stable operation based on the clock edge. The JK Flip-Flop is triggered by either the rising or falling edge of the clock, depending on the design. Overall, its versatility, elimination of the invalid state, and reliable toggle operation make the JK Flip-Flop a fundamental component in digital system design.

Theory:

A MOD-N counter is a type of sequential counter that counts a specific number of clock pulses before resetting to zero. The "N" in MOD-N refers to the number of unique states the counter goes through. For example, a MOD-4 counter counts from 0 to 3, then resets back to 0 on the next clock pulse.

MOD-N counters can be built using flip-flops, with the number of flip-flops determining the maximum count value. A 4-bit counter, for instance, can count up to 24 = 16 states, so the maximum modulus it can support is 16. To create a MOD-N counter (where N = 16), logic is added to reset the counter once it reaches N-1.

There are two main types of counters:

* Synchronous Counter: All flip-flops are triggered by the same clock signal, leading to faster and more reliable operation.
* Asynchronous Counter: Flip-flops are triggered one after another, causing propagation delay and timing issues.

The MOD-N counter is widely used in digital systems for applications like frequency division, event counting, and state machine sequencing. Its flexibility and predictable behavior make it an essential component in digital electronics.

Verilog Code:
module mod_n_counter(
input clk,	// Clock signal
input rst_n,	// Asynchronous reset (active low) input [3:0] N,		// Modulus value (up to 15) output reg [3:0] count // 4-bit counter output
);

always @(posedge clk or negedge rst_n) begin if (~rst_n)	// Active low reset
  count <= 4'b0000; // Reset counter to 0 else if (count == N-1)

  count <= 4'b0000; // Reset counter when it reaches N-1 else
count <= count + 1; // Increment counter
end

Experiment 9: Common Source Amplifier with PMOS Current Mirror Load.
Aim: a) To Construct the schematic of Common Source Amplifier with PMOS Current Mirror Load and find its transient response and AC response, by measuring the Unit Gain Bandwidth (UGB), amplification factor by varying transistor geometries, study of the impact of variation in width to UGB. b) Creating the Layout of common source amplifier, using optimum layout methods. And verification for DRC & LVS, extraction of parasitic and performing post layout.
Theory:
A common source amplifier with a PMOS current mirror load is a fundamental analog circuit that provides high voltage gain. In this configuration, an NMOS transistor functions as the main amplifying device, with its gate receiving the input signal and its source connected to ground. The drain of the NMOS is connected to the drain of a PMOS transistor, which forms the output node. Two PMOS transistors are used in a current mirror configuration, where one is diode- connected and biased using an external voltage to set a reference current. This reference is mirrored to the other PMOS transistor, which acts as an active load for the NMOS. As the input voltage increases, the NMOS conducts more current, causing a voltage drop at the output node due to the constant current from the PMOS load. This results in an amplified and inverted output signal. The PMOS current mirror provides a high output resistance, enhancing the overall gain of the amplifier and making it efficient for analog signal processing in CMOS circuits.
